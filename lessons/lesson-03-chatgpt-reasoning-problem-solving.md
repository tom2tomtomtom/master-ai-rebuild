**Advanced Pattern: Pre-Mortem Analysis**

Instead of asking "what could go wrong?", do a pre-mortem:

```

2. Working backwards: What were the warning signs we should have seen?
3. Root cause: What wrong assumption or oversight led to this?
4. Could we have predicted this? What data would have revealed the risk?

Then: What can we do NOW to prevent each failure mode?
```

This reveals risks traditional analysis misses.

**Expert-Level Pattern: Systems Thinking**

For truly complex problems, use systems thinking:

```
Apply systems thinking to this problem:

PROBLEM: [Your challenge]

SYSTEM MAP:
1. Identify key components: What are the main elements in this system?
2. Map relationships: How do these components affect each other?
3. Find feedback loops: Where do cause-and-effect cycle back?
4. Spot leverage points: Where can small changes create big impacts?

ANALYSIS:
- What are the unintended consequences of each solution?
- Where might "fixing" one problem create new problems?
- What are the non-obvious ripple effects?
- How does this interact with other company systems (sales, product, ops)?

SECOND-ORDER THINKING:
- If we do X, then Y happens
- If Y happens, then Z happens
- If Z happens, what does that mean for our original problem?

Draw out the system dynamics. Where's the real leverage?
```

This prevents "whack-a-mole" problem solving where you create new issues.
